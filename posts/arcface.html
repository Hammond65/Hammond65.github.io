Face Recognition

ArcFace: Additive Angular Margin Loss for Deep Face Recognition

March 10, 2026 · Deng et al. (CVPR 2019)

Motivation: The Identity Drift Problem

In our project, we use MimicMotion (based on Stable Video Diffusion) to generate videos from a single reference image with a sequence of poses. The problem: MimicMotion lacks "face identity" awareness. Frame 1 looks correct, but by frame 10 the nose shape changes, and by frame 20 the face looks like a different person entirely.

To solve this identity drift, we need two capabilities: (1) a way to represent a face as numbers — a compact numerical representation called an embedding that captures identity, and (2) a way to compare two face representations and get a similarity score. ArcFace provides both.

Face Recognition Background

A Brief History

Face recognition has evolved dramatically over the decades:

  • Eigenfaces (1991): Used PCA to reduce face images to lower-dimensional space. Simple but not robust to lighting or pose changes.
  • Fisherfaces (1997): Improved upon Eigenfaces using LDA for better class separation.
  • DeepFace (Facebook, 2014): One of the first deep learning approaches — a 9-layer CNN trained on 4M face images achieving 97.35% on LFW, near human-level (97.53%).
  • FaceNet (Google, 2015): Introduced triplet loss for learning face embeddings directly in Euclidean space.

The Face Recognition Pipeline

  1. Face Detection: Locate the face in the image (MTCNN, RetinaFace)
  2. Face Alignment: Detect 5 facial landmarks and apply similarity transformation to standardize position
  3. Feature Extraction: CNN processes the aligned face to produce a 512-dimensional embedding — this is where ArcFace contributes
  4. Comparison: Calculate cosine similarity between embeddings to determine if two faces belong to the same person

What is a Face Embedding?

A face embedding is a vector of 512 numbers that represents a face's identity — think of it as a numerical fingerprint. It captures identity-related features (face shape, eye spacing, nose size, jawline) and ignores non-identity variations (expression, lighting, angle).

From Triplet Loss to SoftMax-Based Methods

Triplet loss was powerful but problematic. It requires carefully selecting triplets (anchor, positive, negative), and most random triplets are "too easy" — the model already gets them right. Hard negative mining is needed but expensive, and training convergence is slow.

Researchers shifted to treating each person as a class and training with SoftMax classification. This is simpler and more stable, but introduces its own problems.

Why SoftMax Falls Short

  1. No explicit margin between classes. SoftMax only requires the correct class score to be the highest. A score barely above the second-best still yields low loss. This doesn't create well-separated embeddings.
  2. Magnitude vs. direction ambiguity. The dot product WTx depends on both magnitude (length) and direction (angle). A sample could get a high score just because its feature vector is long, not because it points in the right direction.

The Foundation: Feature and Weight Normalization

The original logit is WjTx = ||Wj|| · ||x|| · cos(θj), which depends on three factors. We only want the angle to matter.

  • Step 1: Normalize weights — Force ||Wj|| = 1 for every class
  • Step 2: Normalize features — Force ||x|| = 1, then re-scale by a learnable parameter s

After normalization: WjTx = cos(θj). The logit is now purely the cosine of the angle between the feature and the class weight. Only direction matters.

The Core: Angular Margin

The key insight of ArcFace is requiring a safety gap between the correct class and all wrong classes. Think of two students preparing for an exam — Student A trains to pass at exactly 50% (minimum passing score), while Student B trains to pass at 80%. On exam day, Student B is far more robust.

ArcFace adds an angular margin m to the angle between the feature and its correct class weight:

L = -log(es·cos(θy + m) / (es·cos(θy + m) + Σ es·cos(θj)))

By adding m to the angle, the correct class must have an even smaller angle (be more aligned) to achieve the same logit value. This forces the network to learn more discriminative features.

Training Setup

  • Input: 112×112 aligned face images
  • Backbone: ResNet-100 producing 512-d features
  • Dataset: MS1MV2 with 85,742 identities
  • Output: 512-dimensional face embedding for each input face

Conclusion

ArcFace provides a clean, geometrically interpretable loss function that creates highly discriminative face embeddings. By operating purely in angular space and enforcing an explicit margin, it achieves state-of-the-art face recognition performance. For our project, ArcFace gives us the tool to measure and enforce facial identity consistency across generated video frames — a critical component for solving the identity drift problem in MimicMotion.
