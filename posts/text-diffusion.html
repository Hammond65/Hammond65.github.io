Text-Guided Image Generation with Prompt-to-Prompt Editing

February 15, 2026 · Hertz et al. (ICLR 2023)

Motivation

In previous discussions, we covered how diffusion models denoise step by step — taking a noisy image xₜ and timestep t, predicting noise εθ. We briefly mentioned cross-attention as the "bridge" for text conditioning. This post goes full depth into how and why text can guide image generation, covering CLIP, cross-attention mechanics, DAAM visualization, and Prompt-to-Prompt editing.

CLIP: Turning Words into Useful Numbers

The Problem Before CLIP

Neural networks only understand numbers, not words. We need to convert "A cute cat on grass" into a matrix of numbers that carry visual meaning. Simple approaches like one-hot encoding fail: the distance between "cat" and "dog" is the same as between "cat" and "airplane" — the model can't learn that semantically similar concepts should be close together.

How CLIP Works

CLIP (Contrastive Language-Image Pre-training) was trained by OpenAI on 400 million image-text pairs. It has two encoders trained together:

  • Image Encoder (ViT): image → vector
  • Text Encoder (Transformer): sentence → vector

Training uses contrastive learning: take a batch of 32,768 image-text pairs, maximize similarity for matching pairs while minimizing it for non-matching ones. CLIP uses cosine similarity (measuring angle between vectors) rather than Euclidean distance, scaled by a learnable temperature parameter.

The CLIP Text Encoder Output

In diffusion models, we only use the Text Encoder (the Image Encoder is discarded). The process: tokenize the prompt into 77 tokens → embed each token as a 768-d vector → pass through 12 Transformer layers where tokens interact through self-attention. "Cute" learns it describes "cat," not "grass." The result is a 77×768 matrix where each token carries context-aware semantic meaning.

Freezing CLIP is smart — it was already trained on 400M pairs and understands visual concepts. Training from scratch would require much more data and time, and poor early representations could hurt diffusion model training.

Cross-Attention: The Full Depth

Cross-attention is the mechanism by which each pixel "asks" the text tokens: "Which words are relevant to me?"

Step 1: Creating Q, K, V

  • Query (Q): From image features — each pixel creates a "question vector" encoding what information it needs
  • Key (K): From text features — each word creates an "identity tag"
  • Value (V): From text features — each word creates a "content package" with the actual information to transfer

Step 2: Computing Similarity

S = Q · KT, producing a matrix of shape H×W×77. Each entry S[i,j] is the dot product between pixel i's query and word j's key. High value means they're relevant to each other.

Step 3: SoftMax Normalization

A = softmax(QKT/√d). Dividing by √d prevents extremely large values that cause vanishing gradients. SoftMax converts raw scores into probability weights that sum to 1 for each pixel.

Step 4: Collecting Information

Output = A · V. For a center pixel in a "cute cat on grass" image:

Outputcenter = 0.01·V"A" + 0.04·V"cute" + 0.82·V"cat" + 0.02·V"on" + 0.11·V"grass"

The center pixel now carries 82% "cat" information. The correct mapping is learned purely from the denoising loss — no explicit labels needed.

Multi-Head Attention

One attention head learns one type of relationship. But text-to-image has many: object identity ("cat" maps to cat pixels), color ("red" maps to red regions), spatial relations ("on" connects objects). 8 parallel attention heads each specialize in different relationship types.

DAAM: Opening the Black Box

DAAM (Diffusion Attentive Attribution Maps) aggregates ~2,400 attention maps (from multiple layers, timesteps, and heads) into one clean heatmap per word.

The process: (1) Extract attention maps at every timestep and cross-attention layer, (2) Upscale all maps to the same resolution, (3) Average across all timesteps, layers, and heads, (4) Normalize to [0,1].

Key findings from DAAM:

  • Heatmaps correctly show spatial attribution (e.g., "dog" lights up over the dog region, "book" over the book)
  • Adjectives bind to their nouns ("fluffy" overlaps with "cat")
  • Verbs activate interaction regions ("reading" lights up between dog and book)
  • Early timesteps have coarse attention (global structure), late timesteps have sharp attention (fine details)

Prompt-to-Prompt: Text-Only Image Editing

The core problem: even a small modification of the text prompt leads to a completely different image. Prompt-to-Prompt solves this by manipulating cross-attention maps during generation.

The Algorithm

  1. Run denoising for both source and target prompts simultaneously (same initial noise)
  2. At each timestep: extract attention maps from both, compute an edited map, and override the target's attention

Word Swap

Generate "a cat sitting on a bench" → save all attention maps. Change to "a dog sitting on a bench" → force the model to reuse the saved spatial layout maps. Result: a dog appears in the exact same pose and position as the cat. Proof: the attention map determines spatial layout.

Attention Re-weighting

Multiply the attention column for a specific word by a scalar. For "a fluffy cat" — increasing the weight of "fluffy" makes the cat increasingly fluffy. The token position is deterministic (always at the same index), so we can precisely target individual words.

Conclusion

This chain of technologies — CLIP for encoding text into visually meaningful representations, cross-attention for fusing text with image features, DAAM for visualization and validation, and Prompt-to-Prompt for practical editing — demonstrates that text genuinely controls pixel generation through attention mechanisms. P2P shows that by injecting or manipulating attention maps, we can achieve intuitive, text-only image editing without masks, fine-tuning, or optimization.
