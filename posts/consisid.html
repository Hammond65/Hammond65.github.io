ConsisID: Identity-Preserving Text-to-Video Generation by Frequency Decomposition

February 20, 2026 · Yuan et al. (CVPR 2025)

Motivation

In previous work, we explored MimicMotion (pose-guided video generation) and ArcFace (face recognition). MimicMotion uses a reference person image and pose sequence to generate video — the pose controls body movement, but keeping the face consistent across frames remains a challenge.

Identity-Preserving Text-to-Video (IPT2V) models aim to solve this: given a single reference photo of a person plus a text prompt, generate a video where the person's face stays consistent with the reference while the video content follows the text description. This is harder than normal text-to-video because the model must simultaneously follow text instructions AND maintain pixel-level facial consistency.

The Problem: U-Net vs. DiT

Most existing tuning-free IPT2V methods are built on U-Net architectures (e.g., ID-Animator). But the field is moving to DiT (Diffusion Transformer), and existing U-Net methods cannot be directly adapted.

Why does U-Net work well for identity preservation? U-Net has an encoder-decoder structure with skip connections that naturally preserve spatial detail at multiple scales. DiT lacks these built-in multi-scale connections, making identity injection more challenging.

Key Findings About Frequency

  1. Low-frequency features (face shape, overall contours, proportions) are essential for pixel-level prediction tasks in diffusion models. They help the model converge faster.
  2. Transformers have limited perception of high-frequency information (eye texture, lip shape, skin texture, wrinkles) — the intrinsic identity markers that distinguish one person from another.

ConsisID Framework

ConsisID (Consistent Identity) is built on top of CogVideoX-5B, a DiT-based text-to-video model. The core idea: decompose facial identity into low-frequency and high-frequency signals, then inject each into the right place in DiT.

Global Facial Extractor (Low Frequency)

Goal: provide low-frequency facial information to help DiT converge faster. The approach: concatenate the reference face image with the noise input through the VAE encoder. But there's a subtlety — the reference face contains both high-frequency details and low-frequency structure. The VAE naturally acts as a low-pass filter, preserving the large-scale face structure while smoothing out fine details.

The loss function becomes: Lb = E[||ε - εθ(x₀, t, τθ(y), ψθ(f))||²], where ψθ is the VAE encoding the reference face. Ablation studies show that when this component is removed, the model fails to converge properly.

Local Facial Extractor (High Frequency)

Goal: capture intrinsic identity features that the Global Extractor misses — features that stay the same regardless of expression, pose, age, or makeup. Think of the unique shape of your eyes, the specific curve of your nose, the texture pattern of your skin.

This extractor combines two complementary feature sources:

  • ArcFace features: Identity-discriminative 512-d embeddings trained specifically for face recognition
  • CLIP features: Rich semantic visual features with broader understanding

These are fused using a Q-Former (from BLIP-2) — a module that uses learnable queries to attend to both feature sets, generating identity features enriched with semantic information. The fused features are then injected into the DiT blocks.

Related Work Context

Tuning-Based Methods

Methods like DreamBooth fine-tune the entire network for each new person (1+ hour per person). LoRA trains only a small subset of parameters but still requires per-person optimization.

Tuning-Free Methods

For images: IP-Adapter uses CLIP features through cross-attention; InstantID replaces CLIP with ArcFace features and adds a pose network. For videos: ID-Animator was the only open-source option before ConsisID, but it was limited to U-Net architectures.

Conclusion

ConsisID addresses a fundamental gap in the field: how to preserve facial identity in DiT-based video generation models. By decomposing identity into frequency components and using specialized extractors for each, it achieves state-of-the-art identity preservation without per-person fine-tuning. The combination of ArcFace for discriminative identity features and CLIP for semantic features, fused through Q-Former, provides a robust way to inject identity information into the generation process.
